# ☕ Виртуальное 3D-кафе

Браузерное 3D-кафе (A-Frame) с голосовым чатом через WebRTC (proximity voice — чем ближе
собеседник, тем громче), текстовым чатом, VIP-комнатами, сценой караоке — и приглашением
друзей через Telegram-бота.

## Возможности
- 🚶 Перемещение по кафе (WASD + мышь)
- 🎙️ Голосовой чат в реальном времени, громкость зависит от расстояния между аватарами
- 💬 Текстовый чат
- 🎨 Выбор цвета аватара и имени
- 🔑 Комнаты по коду (создать / войти)
- 🛋️ VIP-комнаты и сцена караоке
- ✈️ Кнопка «Пригласить в Telegram» — бот шлёт сообщение с кодом комнаты и ссылкой

## Стек
- Backend: Node.js, Express, Socket.io (позиции игроков, чат), `peer` (сигнальный сервер PeerJS)
- Frontend: A-Frame (3D), PeerJS (голос через WebRTC)
- Приглашения: Telegram Bot API (`sendMessage`)

## Запуск локально

```bash
npm install
cp .env.example .env
# заполните .env (см. ниже, как получить токен бота и chat_id)
npm start
```

Откройте `http://localhost:3000`.

## Как получить TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID

1. В Telegram напишите **@BotFather** → `/newbot` → следуйте инструкциям → получите токен вида
   `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Вставьте его в `.env` как `TELEGRAM_BOT_TOKEN`.
2. Напишите вашему новому боту любое сообщение (или добавьте его в группу/канал и дайте
   права писать сообщения).
3. Откройте в браузере:
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
4. В ответе найдите `"chat":{"id": ...}` — это и есть `TELEGRAM_CHAT_ID` (для личного чата
   будет положительное число, для группы — отрицательное).
5. Впишите его в `.env`.

Если получатель приглашения каждый раз разный — в приложении в окне «Пригласить в Telegram»
можно ввести конкретный `chat_id` вручную, тогда он переопределит значение из `.env`.

## Деплой на Render

1. Залейте проект в GitHub-репозиторий.
2. На [render.com](https://render.com) → **New → Web Service** → подключите репозиторий.
3. Настройки:
   - Build Command: `npm install`
   - Start Command: `npm start`
4. В разделе **Environment** добавьте переменные из `.env` (`TELEGRAM_BOT_TOKEN`,
   `TELEGRAM_CHAT_ID`, и по желанию `PUBLIC_URL` — url, который выдаст Render, например
   `https://vr-cafe.onrender.com`).
5. Деплой — и ссылка-приглашение в Telegram-сообщениях будет вести на этот адрес.

> Важно для голоса: PeerJS/WebRTC требует HTTPS в проде (Render это даёт из коробки) —
> локально по `http://localhost` браузеры разрешают микрофон без HTTPS, но при деплое
> на собственный сервер (например, Kamatera) нужен SSL-сертификат (Let's Encrypt/Cloudflare).

## Структура

```
vr-cafe/
├── server.js          # Express + Socket.io + PeerJS сервер + /api/invite (Telegram)
├── package.json
├── .env.example
└── public/
    ├── index.html     # экран входа + A-Frame сцена кафе
    ├── style.css       # оформление интерфейса
    └── app.js          # логика: комнаты, голос, чат, движение, приглашение
```
